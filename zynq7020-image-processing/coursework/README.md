## 证据目录

证据统一放在 [`evidence`](evidence/README.md)。每份证据都应能回答五件事：

1. 使用了哪个分支和提交。
2. 运行了什么命令或现场步骤。
3. 得到了什么结果。
4. 结果来自远程仿真/构建还是现场开发板。
5. 还有什么遗留问题。

不要把未经真实运行确认的截图、日志或结论写成“已通过”。没有开发板的阶段只能写“远程仿真和构建通过，待现场验证”。

失败时也要保存材料：执行步骤、完整错误文本、最后一个正常现象、实际板卡连接和工具版本。

## Git 使用约定

- `upstream` 是教师仓库，只拉取不推送。
- `origin` 是个人仓库，保存个人提交和分支。
- `main` 只合并已整理、可复现、边界清楚的阶段成果。
- 实验分支使用 `exp/NN-name`；大拓展继续留在 `exp/05-ext-scaling`，等现场验证后再合并。
- 详细历史记录见 [`docs/REMOTE_PROGRESS_LOG.md`](docs/REMOTE_PROGRESS_LOG.md)。
