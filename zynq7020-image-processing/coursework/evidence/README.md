## 单次记录格式

单次记录建议使用 Markdown，至少写明：

1. 日期、分支和提交号。
2. 工具版本、板卡或仿真环境。
3. 执行命令或现场操作步骤。
4. 关键结果、截图或日志摘要。
5. 结论、遗留问题和下一步。

截图命名采用 `expNN_现象_参数.ext`，例如：

```text
exp00_wave_edge_frame_done.png
exp04_hdmi_threshold80.jpg
exp05_mode_overlay_threshold40.jpg
ext_letterbox_portrait_compare.png
```

## 现场证据原则

- HDMI 照片应能看清有效画面；必要时再补一张包含开发板和显示器的照片。
- 串口日志优先保存原始文本，不只保存截图。
- 资源、时序和 DRC 摘要应标注 Vivado 版本、分支和提交。
- 失败记录同样要归档，必须包含完整错误文本和最后一个正常现象。
